"Fase 10 — Recursos finais

"Implemente os últimos módulos da plataforma:
1. NPS AUTOMÁTICO AO ENCERRAR PROJETO:
Quando o status de um projeto é alterado para 'Encerrado', o sistema aguarda 1 hora e então dispara automaticamente um e-mail via Resend para todos os clientes vinculados ao projeto com link único para responder a avaliação.
Implementar via Supabase Edge Function send-nps-email agendada com pg_cron ou disparada por trigger na alteração de status.
O link de avaliação é único por cliente por projeto — gerado com token UUID armazenado na tabela nps_responses:
sqlCREATE TABLE IF NOT EXISTS nps_responses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  client_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  token TEXT UNIQUE NOT NULL DEFAULT gen_random_uuid()::text,
  nps_score INTEGER CHECK (nps_score BETWEEN 0 AND 10),
  satisfaction_stars INTEGER CHECK (satisfaction_stars BETWEEN 1 AND 5),
  prazo_resposta TEXT CHECK (prazo_resposta IN ('no_prazo','pequeno_atraso','atraso_significativo')),
  comentario TEXT,
  respondido BOOLEAN DEFAULT false,
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '15 days'),
  responded_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
RLS: admin e gerente responsável leem; cliente insere apenas via token público (sem autenticação).
Formulário público de avaliação — rota /avaliacao/[token] (sem necessidade de login):

Design com identidade visual Clarifyse (fundo azul escuro, card centralizado, logo)
Pergunta 1 — NPS: 'Em uma escala de 0 a 10, o quanto você recomendaria a Clarifyse para um colega ou parceiro de negócio?' — escala visual clicável de 0 a 10 com rótulos nas extremidades. Classificação automática: 0-6=Detrator | 7-8=Neutro | 9-10=Promotor
Pergunta 2 — Qualidade: 'Como você avalia a qualidade do entregável final?' — escala de 1 a 5 estrelas clicáveis
Pergunta 3 — Prazo: 'O projeto foi entregue dentro do prazo acordado?' — três opções: 'Sim, dentro do prazo' / 'Com pequeno atraso' / 'Com atraso significativo'
Pergunta 4 — Comentário (opcional): campo de texto livre
Botão 'Enviar Avaliação'
Após envio: 'Obrigado pelo seu feedback! Ele é fundamental para continuarmos evoluindo. — Equipe Clarifyse'

3. LOG DE ATIVIDADES DO SISTEMA (somente administrador):
Aba 'Log de Atividades' dentro de Configurações.
Label: 'AUDITORIA' | Título: 'Log de Atividades'
sqlCREATE TABLE activity_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE SET NULL,
  user_name TEXT NOT NULL,
  user_role TEXT NOT NULL,
  action TEXT NOT NULL,
  category TEXT NOT NULL CHECK (category IN ('autenticacao','projetos','clientes','gerentes','campo','financeiro','documentos','configuracoes','acessos')),
  context TEXT,
  ip_address TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_activity_logs_created ON activity_logs(created_at DESC);
CREATE INDEX idx_activity_logs_category ON activity_logs(category);
RLS: apenas admin lê. Sistema escreve via service role.
Retenção: 90 dias via pg_cron.
Eventos registrados automaticamente:

Autenticação: login bem-sucedido, login com falha, logout
Projetos: criado, editado, status alterado, duplicado, excluído, restaurado, excluído permanentemente
Clientes: criado, editado, desativado, reativado, excluído, senha redefinida
Gerentes: criado, editado, desativado, reativado, excluído, senha redefinida
Campo: sincronização Google Sheets (sucesso/erro), atualização manual
Financeiro: dados financeiros preenchidos/editados, calculadora utilizada
Documentos: adicionado, removido, visibilidade alterada
Configurações: qualquer alteração nas configurações do sistema
Acessos: cliente acessou o portal, cliente acessou página de projeto, cliente respondeu NPS

Interface: lista cronológica (mais recente primeiro) + paginação (50 por página) + filtros por usuário/categoria/período/projeto + campo de busca livre + botão 'Exportar Log (Excel)'.

4. PARCEIROS DE PAINEL (somente administrador — Configurações → Parceiros):
sqlCREATE TABLE panel_partners (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nome TEXT NOT NULL,
  site TEXT,
  email_contato TEXT,
  telefone TEXT,
  notas TEXT,
  status TEXT DEFAULT 'ativo' CHECK (status IN ('ativo','inativo')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE TABLE panel_partner_cpr (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id UUID REFERENCES panel_partners(id) ON DELETE CASCADE,
  perfil TEXT NOT NULL,
  faixa_perguntas TEXT NOT NULL,
  cpr_valor DECIMAL(8,2),
  data_cotacao DATE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE TABLE panel_partner_reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id UUID REFERENCES panel_partners(id) ON DELETE CASCADE,
  project_id UUID REFERENCES projects(id) ON DELETE SET NULL,
  qualidade_amostral INTEGER CHECK (qualidade_amostral BETWEEN 1 AND 5),
  cumprimento_prazo INTEGER CHECK (cumprimento_prazo BETWEEN 1 AND 5),
  custo_beneficio INTEGER CHECK (custo_beneficio BETWEEN 1 AND 5),
  comentario TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
RLS: apenas admin lê e escreve em todas as tabelas de parceiros.
Interface:

Lista de parceiros: Nome | Site | E-mail | Telefone | Status | Ações
Cadastrar parceiro: Nome, Site, E-mail, Telefone, Notas, Status
Tabela de CPR por perfil editável para cada parceiro (8 combinações: 4 perfis × 3 faixas de perguntas). Badge '⚠️ Cotação desatualizada' quando data > 90 dias
Avaliação por projeto: vincula parceiro a um projeto + registra qualidade amostral (1-5⭐) + cumprimento de prazo (1-5⭐) + custo-benefício (1-5⭐) + comentário
Painel comparativo: tabela com CPR médio de todos os parceiros ativos por perfil lado a lado
Integração com Calculadora: quando o admin abre a calculadora e seleciona um perfil, o sistema busca a média dos CPRs dos parceiros ativos para aquele perfil e exibe automaticamente no campo CPR com a indicação 'Média dos parceiros: R$ XX,XX'
