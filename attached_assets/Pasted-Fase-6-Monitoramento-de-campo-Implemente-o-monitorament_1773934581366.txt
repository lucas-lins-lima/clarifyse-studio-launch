Fase 6 — Monitoramento de campo

"Implemente o monitoramento de campo completo dentro de cada projeto:
CONFIGURAÇÃO DE CAMPO (admin e gerente — aba 'Campo' dentro do projeto):
Seção 'Métricas Gerais':

Meta Total de Entrevistados (número)
Tempo Médio Esperado de Entrevista (em minutos)

Seção 'Cotas' — botão 'Adicionar Cota' abre modal para escolher o tipo:
TIPO 1 — MAPEAMENTO NUMÉRICO (ex: Gênero, Tipo de Usuário):

Nome da Cota
Botão 'Adicionar Subcategoria' — para cada uma: Rótulo de exibição + Valor numérico correspondente na planilha + Meta (número de respondentes)
Exemplo: Gênero → 1=Masculino(meta:150) | 2=Feminino(meta:150) | 3=Não-Binário(meta:0)

TIPO 2 — FAIXAS ETÁRIAS:

Nome da Cota
Botão 'Adicionar Faixa' — para cada uma: Rótulo + Valor mínimo + Valor máximo + Meta
Faixas padrão sugeridas ao criar (editáveis): 16-17 | 18-24 | 25-34 | 35-44 | 45-54 | 55-64 | 65+
Sistema classifica automaticamente cada respondente na faixa correta pelo valor numérico da coluna de idade

TIPO 3 — TEXTO/CATEGORIA (ex: Região):

Nome da Cota
Lista de valores esperados com suas metas (ex: 'São Paulo' → meta 200)
Sistema conta ocorrências exatas de cada texto na coluna

TIPO 4 — BOOLEANA (ex: Usuário/Não Usuário):

Nome da Cota
Define qual valor = Sim e qual = Não (ex: 1=Sim, 0=Não)
Meta para cada opção

Cotas reordenáveis via drag-and-drop. Cada cota editável e removível a qualquer momento. Alterações nas cotas reprocessam a contagem sem apagar dados coletados.
INTEGRAÇÃO GOOGLE SHEETS (Opção A — recomendada):
Campo para colar o link da planilha com instrução: 'Cole aqui o link de compartilhamento. Basta compartilhar com Qualquer pessoa com o link pode visualizar.'
Sistema converte automaticamente qualquer URL do Google Sheets para o formato CSV:
https://docs.google.com/spreadsheets/d/[ID]/export?format=csv&gid=[GID]
Botão 'Carregar Planilha':

Chama Edge Function que busca a planilha e lê a primeira linha (cabeçalho)
Exibe lista de todas as colunas detectadas com nomes exatos (ex: Coluna A: 'ID', Coluna B: 'Genero', etc.)
Para cada cota configurada, admin seleciona via dropdown qual coluna corresponde
Para Tempo Médio de Entrevista, seleciona a coluna correspondente
Admin confirma qual coluna usar como referência para contar linhas válidas (total de entrevistados)
Sistema salva esse mapeamento coluna ↔ cota

Sincronização automática:

OBRIGATÓRIO: rodar via Supabase Edge Function agendada com pg_cron a cada 5 minutos — o frontend NUNCA chama a planilha diretamente
Frontend apenas lê o dado já sincronizado no banco
Cada sincronização registra timestamp na tabela field_sync_log
Erro de sincronização: salvar mensagem de erro no banco → admin vê alerta vermelho com descrição técnica → cliente vê apenas 'Dados sendo atualizados...'
Botão manual 'Atualizar Agora' disponível para admin e cliente (chama a Edge Function sob demanda)

ENTRADA MANUAL (Opção B):

Botão 'Atualizar Dados de Campo' dentro do projeto
Formulário com: Total de entrevistados realizados + Tempo médio (min) + Valor realizado de cada subcategoria de cada cota
Salva com data/hora da última atualização manual

Banco de dados — novas tabelas:
sqlCREATE TABLE field_config (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  meta_total INTEGER,
  tempo_medio_esperado INTEGER,
  integration_mode TEXT CHECK (integration_mode IN ('sheets', 'manual')),
  sheets_url TEXT,
  sheets_csv_url TEXT,
  reference_column TEXT,
  time_column TEXT,
  last_sync_at TIMESTAMPTZ,
  last_sync_error TEXT,
  realizado_total INTEGER DEFAULT 0,
  tempo_medio_real DECIMAL(5,2),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE field_quotas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  nome TEXT NOT NULL,
  tipo TEXT CHECK (tipo IN ('numerico','faixa_etaria','texto','booleano')),
  coluna_planilha TEXT,
  ordem INTEGER NOT NULL,
  config JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE field_quota_results (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  quota_id UUID REFERENCES field_quotas(id) ON DELETE CASCADE,
  subcategoria TEXT NOT NULL,
  meta INTEGER DEFAULT 0,
  realizado INTEGER DEFAULT 0,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE field_sync_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
  status TEXT CHECK (status IN ('success','error')),
  rows_count INTEGER,
  error_message TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
RLS: admin e gerente responsável leem e escrevem; cliente lê apenas dados dos seus projetos.
VISUALIZAÇÃO DO CLIENTE (Seção 'Monitoramento de Campo' na página do projeto):
Exibida apenas se status for 'Campo' ou se já houver dados de campo. Caso contrário: 'O monitoramento de campo estará disponível quando as coletas iniciarem.'
Label: 'MONITORAMENTO DE CAMPO' | Título: 'Acompanhamento em Tempo Real'
4 cards de métricas no topo:

Card 1 — Total de Entrevistados: número grande + 'de X entrevistados' (meta) + barra de progresso teal→roxo + percentual
Card 2 — Tempo Médio: número grande em minutos + 'Esperado: X min'
Card 3 — Status do Campo: badge 'Em Andamento' (verde pulsante) / 'Encerrado' / 'Aguardando'
Card 4 — Última Atualização: data/hora (horário de Brasília) + botão 'Atualizar agora'

Seção de Cotas — para cada cota, painel dedicado com gráfico de barras horizontais (Recharts):

Mapeamento Numérico: para cada subcategoria → nome + contador 'X / Y' + barra de progresso individual + percentual + badge (Meta atingida=verde / Em andamento=teal / Abaixo do esperado=amarelo)
Faixas Etárias: mesmo layout com rótulo de cada faixa
Texto/Categoria: lista com barras por categoria
Booleana: dois cards lado a lado

Nota de transparência no rodapé: 'Os dados desta seção são atualizados periodicamente a partir das coletas em andamento. Qualquer dúvida sobre o progresso do campo, entre em contato com o gerente do seu projeto.'
Performance:

Componentes de gráfico (Recharts) carregados com React.lazy
React Query para cache dos dados de campo com revalidação a cada 5 minutos no frontend"
